## Вход в систему

1. Перейдите на страницу входа: `/auth/login`
2. Введите учетные данные менеджера:
   - **Логин:** `manager`
   - **Пароль:** `manager123`
3. После успешного входа вы будете перенаправлены в панель управления

**Важно:** Менеджер имеет доступ к управлению товарами, заказами, блогом и обращениям, но не может управлять пользователями, категориями и системным контентом.

---

## Дашборд

Дашборд (`/admin/`) - главная страница панели управления, отображающая:

- **Статистику:**
  - Количество товаров
  - Количество заказов
  - Заказы, ожидающие обработки
  - Количество статей в блоге
  - Новые обращения

- **Последние заказы:** Список из 10 последних заказов с возможностью перехода к деталям

**Примечание:** Статистика по пользователям и контенту доступна только администраторам.

## Управление товарами

### Создание товара

1. Перейдите в **Товары и заказы** → **Товары** (`/admin/products`)
2. Нажмите **Добавить**
3. Заполните форму:
   - **Название** - название товара
   - **Slug** - URL-дружественный идентификатор (генерируется автоматически)
   - **Описание** - полное описание товара
   - **Краткое описание** - краткое описание для карточек товаров
   - **Цена** - цена в рублях
   - **Количество на складе** - остаток товара
   - **Категория** - выбор категории из списка
   - **Изображение:**
     - Загрузка файла (PNG, JPG, JPEG, GIF, WEBP)
     - Или указание URL изображения
     - Приоритет у загруженного файла
   - **Активен** - отображать товар в каталоге
4. Нажмите **Сохранить**

### Редактирование товара

1. В списке товаров нажмите **Редактировать** рядом с нужным товаром
2. Измените нужные поля:
   - Обновление цены
   - Изменение количества на складе
   - Редактирование описания
   - Замена изображения
3. Нажмите **Сохранить**

**Важно:** Менеджер может редактировать товары, но не может их удалять. Удаление доступно только администраторам.

### Обновление количества товара

1. Откройте редактирование товара
2. Измените значение в поле **Количество на складе**
3. Нажмите **Сохранить**

**Совет:** Регулярно проверяйте остатки товаров и обновляйте их количество.

## Управление заказами

### Просмотр списка заказов

1. Перейдите в **Товары и заказы** → **Заказы** (`/admin/orders`)
2. Отображается список всех заказов:
   - ID заказа
   - Пользователь
   - Сумма заказа
   - Статус
   - Дата создания

### Просмотр деталей заказа

1. В списке заказов нажмите **Подробнее** рядом с нужным заказом
2. Отображается полная информация:
   - **Информация о заказе:**
     - Пользователь
     - Дата создания
     - Сумма заказа
     - Адрес доставки
     - Телефон (если указан)
   - **Товары в заказе:**
     - Название товара
     - Цена за единицу
     - Количество
     - Итоговая стоимость

### Изменение статуса заказа

Менеджер может изменять статусы заказов для отслеживания их обработки:

1. Откройте детали заказа
2. В форме **Изменение статуса** выберите новый статус:
   - **Ожидает** (pending) - заказ создан, ожидает обработки
   - **В обработке** (processing) - заказ обрабатывается
   - **Отправлен** (shipped) - заказ отправлен клиенту
   - **Доставлен** (delivered) - заказ доставлен
   - **Отменен** (cancelled) - заказ отменен
3. Нажмите **Обновить статус**

**Рекомендации:**
- Обновляйте статус заказа сразу после начала обработки
- Уведомляйте клиентов об изменении статуса (вручную)
- Отмечайте заказы как "Доставлен" после подтверждения получения

## Управление блогом

### Создание статьи

1. Перейдите в **Контент (CMS)** → **Блог** (`/admin/blog`)
2. Нажмите **Добавить**
3. Заполните форму:
   - **Название** - заголовок статьи
   - **Slug** - URL-дружественный идентификатор (генерируется автоматически)
   - **Краткое описание** - анонс статьи (отображается в списке статей)
   - **Содержание** - полный текст статьи
   - **Изображение:**
     - Загрузка файла (PNG, JPG, JPEG, GIF, WEBP)
     - Или указание URL изображения
   - **Опубликовано** - отображать статью на сайте
4. Нажмите **Сохранить**

### Редактирование статьи

1. В списке статей нажмите **Редактировать** рядом с нужной статьей
2. Измените нужные поля:
   - Заголовок
   - Содержание
   - Изображение
   - Статус публикации
3. Нажмите **Сохранить**

**Советы:**
- Используйте черновики (снимите галочку "Опубликовано") для сохранения незавершенных статей
- Проверяйте текст на ошибки перед публикацией
- Используйте качественные изображения для привлечения внимания

### Управление публикацией

- **Опубликовать статью:** Установите галочку "Опубликовано" и сохраните
- **Снять с публикации:** Снимите галочку "Опубликовано" и сохраните

## Управление обращениями

### Просмотр обращений

1. Перейдите в раздел **Обращения** (`/admin/messages`)
2. Доступны фильтры:
   - **Все обращения** - показать все сообщения
   - **Только непрочитанные** - показать только новые обращения
3. Отображается информация:
   - Имя отправителя
   - Email
   - Тема сообщения
   - Дата отправки
   - Статус прочтения

### Просмотр деталей обращения

1. В списке обращений нажмите **Подробнее** рядом с нужным обращением
2. Отображается полная информация:
   - Имя отправителя
   - Email
   - Телефон (если указан)
   - Тема сообщения
   - Текст сообщения
   - Дата отправки
3. Обращение автоматически помечается как прочитанное

### Удаление обращения

1. В деталях обращения нажмите **Удалить**
2. Подтвердите удаление

**Рекомендации:**
- Регулярно проверяйте новые обращения
- Удаляйте обработанные обращения для поддержания порядка

---

## Ограничения доступа

Менеджер **не имеет доступа** к следующим разделам:

- Управление пользователями
- Управление категориями
- Управление системным контентом (About, Contact, Hero слайды)

Для выполнения этих действий обратитесь к администратору.
